本週最重要心得：學習迭代過程，重要的是不斷問好問題

這週讀完破繭成蝶，印象最深的部分是「感性與理性的結合」
以中國那邊的說法：互聯網的戰爭已進入下半場，能更專心地專注在產品品質上
但並不是單純的「把產品做好」，而是要更有效率地判斷戰場，專注進攻

以我自己作為產品為例，我的短期戰場是實驗室 App 發展需求，中長期是 Android 開發工程師
開學前兩個月的幾個關鍵成長指標：

  1. Android 開發能力：課程的完成程度（實作量）、Interview Question 的解答分數（實戰問題所需知識）
  2. 課業準備能力：計組修課通過、Deep Learning 預習過一次
  3. 產品執行能力：產品測試（各種文檔的建立）、客服、協助團隊改進產品、語言能力

下一週需滾動調整的部分：計組通過的高效率方式

Day 7~9 | Flora Alarm Journey

發表於 2020-06-21

這幾天比較常與學長姐們交流，所面對的問題也開始立體起來。

實際要解決的問題有：

1. 目標：團隊的 vs. 老師的 vs. 學長姐的 vs. 我的

老師是團隊領導，但目前團隊的能力尚不足以讓老師作顧問或純決策，抑或是說目前的資源也供應不起現有成員全力投入，這樣的狀況下，老師幾乎作為唯一決策點，會導致團隊跟不上市場變化。同時，義路跟振弘的首要目標是畢業，而關於未來的目標，義路感覺是個執行者、振弘則還有需考量的地方，因此關鍵就變成振偉的能力是否足以帶領團隊前進，來決定明年 Flora 能走多遠。

雖感受得到老師想將團隊帶起來，但目前團隊的核心構成元素不是目標與未來前景，而是老師的個人魅力。這本身沒有不好，但在老師還要分心其他事物（且比重越來越重）的狀況下，團隊的決策與執行速度就不足以站穩市場。

以目前的組織編制來看，我不會是 Flora 團隊的核心成員，但鐵定有貢獻心力的機會。不過，比起一開始就大放厥詞，我更需要透過一次次的試煉證明自己，才能爭取到自己在團隊中的位置。以結論而言，至少在開學前，我的目標就只有把技術能力加強到目標水準，同時把被交派的任務做好，切實地幫團隊解決問題而不是製造問題。

2. 客服與 CRM, App Test, 英語和日文能力, Android 開發協助, 學業

上文雖邏輯不夠通順，但事情脈絡已大致理清，我在 9 月開學前要完成的有：

  1. Android 開發協助：協助振弘 Flora Alarm、並為 Flora Alarm 做前置作業

  2. 英文與日文的熟練度：要能翻譯、客服、熟悉市場

  3. 客服與 CRM： 應對風險能力、資料整理、資料庫操作

  4. App Test: 更快速地完成版本測試與更專業的改善建議

  5. 學業：計算機組織、Deep Learning、Group Meeting

一、課程挑選

今天挑選課程時考量的點有：

  1. 是否能達成我的需求：使用 Jetpack Compose 開發、並能將舊 Java 專案轉換過來
  2. 字幕與口音是否易懂：使用倍速播放時能不受語言的限制
  3. 課程是否包含思考框架：希望不僅是學習工具，也從講師的經驗中學習開發方法
  4. 課程內容是否合時宜：雖想法與技術是從過去演化而來，但若會花大量額外時間，則應從現在所需為切入點，之後再視情況往前補，會更有效率。

因此挑了下面兩門課程：

  • The Complete Android 10 & Kotlin Development Masterclass: 以 Kotlin 為主，內容相對新，課程內容較多、有大型專案實做(Trello Clone)，整體學完應能掌握 Kotlin 基礎。

  • The Comprehensive 2020 Android Development Masterclass：以 Java 為主，課程應有超過兩年的歲數（當時最新版本為 Android 7.1），應適合學完現代開發方式後，用來補充 Java 與過去開發方式的課程

而除了上述兩門入門課外，亦有看了幾門未來可能學習的進階課程，都是由同一名講師Vasiliy Zukanov開的、皆獲得很高的評價，以下列做參考：

  • Android Architecture Masterclass
  • Android Unit Testing and Test Driven Development
  • Android Multithreading Masterclass
